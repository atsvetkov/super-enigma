# Transaction Risk Explorer -- Design Spec

## Overview

A single-page interactive web dashboard that lets fraud analysts at RappiMarket visually explore payment transactions across Bogota. The tool replaces CSV-based manual review with coordinated visualizations that make fraud patterns (velocity attacks, geographic clusters, card testing) immediately visible.

**Scope:** Frontend prototype with static synthetic data. No backend, no real API integration, no maps. Core requirements only -- stretch goals deferred.

## Synthetic Data

~1,200 transactions over 5 days. Generated by a one-time script (`scripts/generate-data.ts`) that outputs `src/data/transactions.json`.

### Embedded Fraud Patterns

1. **Velocity attack:** ~20 transactions using 3 different cards, all delivering to the same address ("Calle 85 #15-23"), within a 2-hour lunch window on day 3.
2. **Geographic cluster:** 25+ high-risk (score 70-95) transactions concentrated in "Chapinero" and "Usaquen" during a 4-hour evening window (6pm-10pm) on day 4.
3. **Card testing:** Card `*4532` makes 4 small transactions (15,000-45,000 COP) over 30 minutes, followed by 2 large ones (210,000 and 245,000 COP) 15 minutes later, on day 2.

### Normal Transactions

~85% of the dataset. Characteristics:
- Risk scores 0-45, occasional 45-65
- Spread across all 12-15 neighborhoods
- Amounts 25,000-250,000 COP, normally distributed around 80,000
- Timestamps spread across 7am-11pm with peaks at lunch (12-2pm) and dinner (6-9pm)
- Mix of approved (~88%), declined (~8%), under_review (~4%)

### Neighborhoods

Chapinero, Usaquen, Suba, Kennedy, Teusaquillo, La Candelaria, Engativa, Fontibon, Bosa, Ciudad Bolivar, Rafael Uribe, San Cristobal.

## Layout

Single-page coordinated dashboard:

```
+--------------------------------------------------+
|  Header: "Transaction Risk Explorer"    [metrics] |
+----------+---------------------------------------+
|          |                                       |
| Filters  |   Main Scatter Plot                   |
| Panel    |   (Time x Amount, color = risk)       |
|          |                                       |
| - Date   |                                       |
| - Risk   +-------------------+-------------------+
| - Status | Neighborhood      | Hourly Volume     |
| - Amount | Breakdown         | (bar chart)       |
| - Neigh. | (bar chart)       |                   |
| - Card   +-------------------+-------------------+
|          |                                       |
| Fraud    | Transaction Table                     |
| Alerts   | (sortable, filtered)                  |
|          |                                       |
+----------+---------------------------------------+
```

All components share the same filter state. Changing a filter updates every chart and the table.

## Components

### FilterPanel
Left sidebar. Controls:
- **Date range:** Start/end day selector (5-day range)
- **Risk score:** Range slider (0-100, dual handles)
- **Status:** Multi-select checkboxes (approved/declined/under_review)
- **Amount:** Range slider (0-250,000 COP)
- **Neighborhood:** Multi-select dropdown
- **Card last 4:** Text input for exact match
- **Reset filters** button

### SummaryMetrics
Top bar showing filtered stats:
- Total transactions (filtered / total)
- Average risk score
- Decline rate (%)
- High-risk count (risk > 70)

### ScatterPlot (main visualization)
- X-axis: timestamp
- Y-axis: amount (COP)
- Dot color: risk score (green 0-40, yellow 40-70, red 70-100)
- Hover: tooltip with transaction summary
- Click: opens TransactionDetail modal
- Brush select: updates filters to selected region

### NeighborhoodChart
- Horizontal bar chart
- Each bar = one neighborhood, length = transaction count
- Bar color = average risk score for that neighborhood
- Click a bar: filters to that neighborhood

### HourlyVolumeChart
- Bar chart: X = hour of day (0-23), Y = transaction count
- Bars colored by average risk score for that hour
- Hours exceeding 2x average volume highlighted with a red outline as anomaly indicator

### FraudAlerts
- List of pre-computed alerts based on detection logic
- Types: velocity, geographic cluster, card testing
- Each alert shows: type icon, short description, transaction count
- Clicking an alert auto-applies filters to isolate that pattern across all charts

### TransactionTable
- Sortable columns: timestamp, amount, neighborhood, risk score, status, card last 4
- Shows currently filtered transactions
- Click a row to open TransactionDetail

### TransactionDetail
- Modal or slide-out panel
- Shows all fields for a single transaction
- Appears on scatter dot click or table row click

## Filter & Alert Logic

### useFilters hook
- Manages all filter state as a single object
- Provides `setFilter(key, value)` and `resetFilters()`
- Memoizes filtered transaction list
- Exposes `filteredTransactions` array consumed by all components

### useAlerts hook
- Runs detection algorithms against the full dataset on load
- Returns a list of alert objects: `{ type, description, transactionIds }`
- When an alert is clicked, applies appropriate filters

### detection.ts utilities
- `detectVelocity(transactions)`: Groups by delivery address, finds clusters of >5 transactions within 2-hour windows
- `detectGeographicClusters(transactions)`: Finds neighborhoods with >15 high-risk transactions in a 4-hour window
- `detectCardTesting(transactions)`: Finds cards with 3+ small transactions followed by large ones within a short period
